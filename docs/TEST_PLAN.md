### 2.3 System-Type-Specific Behavioral Test Requirements

This project is classified as **Web (Canvas/3D)** (React Three Fiber + Three.js) and **Web (DOM)** (React UI controls).

| System Type | Behavioral Test Must Verify | Probe Method |
|------------|---------------------------|-------------|
| Web (Canvas/3D) | Mouse interaction on canvas produces visible scene change (brick appears/disappears) | Playwright mouse.click at canvas coordinates + pixel comparison or Three.js scene graph inspection |
| Web (DOM) | UI control interaction (button click, color swatch) produces visible DOM change | Playwright interaction + DOM assertion |

**Canvas/3D Behavioral Test Rules:**
- E2E tests MUST verify rendering output using screenshot comparison or Three.js scene graph inspection.
- Tests must use the app's actual interaction model (click at canvas coordinates where bricks render).
- At least one screenshot-based assertion per core user flow.
- Browser console must be checked for WebGL/Three.js errors after each interaction.
- `window.__appState` or similar internal state inspection is NOT sufficient as the sole verification method.

### 2.4 Behavioral Replay Tests

Behavioral replay tests will be auto-generated by the `feature-behavioral-smoke-agent` and `behavioral-validation-agent` from runtime observations and stored at `tests/behavioral/{FR-ID}/`. These run in CI automatically. This test plan acknowledges their existence but does not define them individually.

## 10. Test Maintenance

When adding new tests:
1. Assign a unique Test ID following the `T-<DOMAIN>-<FR_ID>-<SEQ>` convention.
2. Add to the appropriate section in this document.
3. Update the traceability matrix (Section 8).
4. Reference the Test ID in the test code.

When removing tests:
1. Document why in the PR description.
2. Mark as deprecated in this document (do not delete).
3. Update the traceability matrix.

When FRs are modified:
1. Update corresponding test cases to match new acceptance criteria.
2. Update the traceability matrix.
3. Increment the document version.
